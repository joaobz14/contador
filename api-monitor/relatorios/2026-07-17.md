## O que foi tentado (exatamente)

1. **WebFetch** (busca do lado da Anthropic, fora do proxy):
   - `developers.mercadolivre.com.br/devcenter/news/` → **HTTP 403 Forbidden**
   - `developers.mercadolivre.com.br/pt_br/api-docs-pt-br` → **HTTP 403 Forbidden**
   - `open.shopee.com/announcements` → **"unable to fetch from open.shopee.com"** (domínio bloqueado)
   - `open.shopee.com/documents` → **"unable to fetch from open.shopee.com"**
2. **Chromium headless (Playwright)**, através do proxy do agente, com User-Agent de
   navegador real: as 4 URLs deram `net::ERR_TUNNEL_CONNECTION_FAILED`.
3. **curl** via proxy (com o CA bundle correto):
   - ML e Shopee → `CONNECT tunnel failed, response 403`
   - `github.com` (host permitido, controle) → conectou (tunnel OK)
4. **Diagnóstico do proxy** (`/__agentproxy/status`): registrou
   `connect_rejected — gateway answered 403 to CONNECT (policy denial)` para
   `developers.mercadolivre.com.br:443` (e outros hosts fora da allowlist).
   O `README` do proxy orienta: *"403/407 from the proxy ... report the blocked
   host"* — não contornar.

**Conclusão:** o bloqueio é do **ambiente de nuvem**, não das fontes.

## Por que isso NÃO impede a rotina semanal

A tarefa semanal foi configurada para rodar **na máquina Windows do dono**
(`C:\contador`), via `claude -p` local, onde a **rede é aberta**. Lá as 4 fontes
são alcançáveis (sujeitas apenas à bot detection normal dos sites, que um
navegador local costuma passar). O **snapshot inicial (baseline) será criado na
primeira execução local** — o `prompt-semanal.md` já trata "sem snapshot anterior"
como primeira coleta, sem comparação.

## O que eu precisaria (se você quiser o baseline criado a partir da nuvem)

Não é necessário para a operação — mas, se você quisesse capturar o baseline
daqui, seria preciso **liberar esses domínios na política de rede do ambiente de
nuvem** (escolhida na criação do ambiente):
`developers.mercadolivre.com.br`, `open.shopee.com`. Ainda assim, o ML respondeu
403 até ao WebFetch (bot detection), então o caminho **confiável** é mesmo a
execução local no seu Windows.
